Configuración inicial Jetson Nano y LM Studio

Documentación técnica sobre el despliegue inicial de una NVIDIA Jetson Nano. El proyecto abarca desde la conexión de hardware y configuración del sistema operativo (Ubuntu) hasta la implementación de almacenamiento externo y la instalación de software de Inteligencia Artificial Generativa (LM Studio).

Resumen

El objetivo de esta práctica fue poner en marcha un dispositivo de computación en el borde (Edge Computing) para tareas de IA. El proceso comenzó con la preparación del hardware Jetson Nano y su cableado. Posteriormente, se realizó la configuración "out-of-the-box" del sistema operativo, incluyendo localización, red y creación de usuarios administrativos.

Una fase crítica consistió en el montaje técnico de una unidad SSD externa mediante comandos de terminal para expandir el almacenamiento. Finalmente, se procedió a la instalación de las librerías necesarias (libfuse) para ejecutar LM Studio y descargar un modelo de lenguaje (LLM) de 4 billones de parámetros para realizar pruebas de inferencia.

1. Preparación de Hardware y Primer Arranque

Verificación Física y Encendido

Se procedió a la preparación del material, asegurando la disponibilidad de la placa Jetson Nano y su cableado correspondiente. Tras conectar el dispositivo a la corriente y al monitor para verificar su funcionamiento, se inició el asistente de configuración. El primer paso consistió en aceptar las licencias de uso de NVIDIA y seleccionar el idioma del sistema.

5. Preparación del Entorno y Montaje SSD

Configuración de Almacenamiento Externo

Para preparar el entorno de IA, consultamos la guía oficial de NVIDIA Jetson. Identificamos el disco SSD conectado mediante comandos de terminal (lspci, lsblk). Procedimos a formatear la unidad a ext4, crear el directorio de montaje /ssd y montar el dispositivo. Se editó el fichero /etc/fstab para asegurar la persistencia del montaje tras los reinicios y se asignaron los permisos adecuados al usuario.

6. Instalación y Prueba de LM Studio

Despliegue de IA Generativa

Se instalaron las dependencias necesarias (libfuse2) para ejecutar aplicaciones AppImage. Tras otorgar permisos de ejecución al archivo descargado, lanzamos LM Studio. Desde la interfaz, buscamos y descargamos el modelo de lenguaje "qwen2-vl-4b" (4 billones de parámetros). Finalmente, realizamos una prueba de inferencia mediante el chat integrado para verificar el rendimiento en tokens por segundo.
